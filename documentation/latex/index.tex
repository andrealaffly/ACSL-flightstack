\hypertarget{index_autotoc_md1}{}\doxysection{Introduction}\label{index_autotoc_md1}
The {\bfseries{Advanced Control Systems Lab (ACSL) Flight Stack}} is a PX4-\/compatible offboard flight stack designed for multi-\/rotor UAVs (Uncrewed Aerial Vehicles). This software supports the UAV control community by offering a freeware, open-\/source solution that serves as a shared platform, facilitating comparison of research results.

We strongly encourage control practitioners, researchers, and UAV enthusiasts to\+:


\begin{DoxyItemize}
\item {\bfseries{Use this code}}
\item {\bfseries{Improve it}}
\item {\bfseries{Provide constructive feedback}}
\item {\bfseries{Propose edits through \href{https://github.com/andrealaffly/ACSL-flightstack.git}{\texttt{ Git\+Hub}}}}
\end{DoxyItemize}

This flight stack is tailored for autonomous UAVs with collinear propellers, such as quadcopters, X8-\/copters, and hexacopters. Currently, it supports quadcopters and X8-\/copters. However, by modifying the mixer matrix, which defines the relationship between total thrust, moment of thrust, and thrust produced by each motor, this software can be extended to other UAV configurations with collinear propellers.\hypertarget{index_autotoc_md2}{}\doxysection{Outlook on the Control Architecture}\label{index_autotoc_md2}
Autonomous UAVs with collinear propellers are inherently under-\/actuated. For this reason, the software includes\+:


\begin{DoxyItemize}
\item {\bfseries{Inner Loop}}\+: Handles the rotational dynamics.
\item {\bfseries{Outer Loop}}\+: Handles the translational dynamics.
\end{DoxyItemize}

Both loops are governed by nonlinear equations of motion.\hypertarget{index_autotoc_md3}{}\doxysubsection{Available Control Solutions}\label{index_autotoc_md3}
This software currently offers two control solutions for the inner and outer loops\+:


\begin{DoxyEnumerate}
\item {\bfseries{Continuous-\/\+Time Feedback-\/\+Linearizing Control Law}} combined with a {\bfseries{PID (Proportional-\/\+Integral-\/\+Derivative) Control Law}}.
\item The above control law is augmented by a {\bfseries{Robust Model Reference Adaptive Control (MRAC) System}}, incorporating a simplified quadratic-\/in-\/the-\/velocity aerodynamic model.
\end{DoxyEnumerate}

For further details on these control architectures, refer to the following publications\+:


\begin{DoxyItemize}
\item M. Gramuglia, G. M. Kumar, and A. L\textquotesingle{}Afflitto, \href{https://doi.org/10.1109/RoMoCo60539.2024.10604361}{\texttt{ \char`\"{}\+Two-\/\+Layer Adaptive Funnel MRAC with Applications to the Control of Multi-\/\+Rotor UAVs,\char`\"{}}} {\itshape 2024 13th International Workshop on Robot Motion and Control (Ro\+Mo\+Co),} Poznań, Poland, 2024, pp. 31-\/36.
\item M. Gramuglia, G. M. Kumar, and A. L’\+Afflitto, \href{https://doi.org/10.2514/6.2024-0755}{\texttt{ \char`\"{}\+A Hybrid Model Reference Adaptive Control System for Multi-\/\+Rotor Unmanned Aerial Vehicles,\char`\"{}}} {\itshape AIAA SCITECH 2024 Forum, 2024.}
\item E. Lavretsky and K. Wise, $\ast$\char`\"{}\+Robust and Adaptive Control\+: With Aerospace Applications,\char`\"{}$\ast$, London, UK\+: Springer, 2012.
\end{DoxyItemize}

Future versions of the software will include additional control systems.\hypertarget{index_autotoc_md4}{}\doxysection{Hardware Requirements}\label{index_autotoc_md4}
This flight stack is compatible with {\bfseries{\href{https://docs.ros.org/en/foxy/Installation.html}{\texttt{ ROS2 Foxy}}}} or {\bfseries{\href{https://docs.ros.org/en/galactic/Installation.html}{\texttt{ Galactic}}}} and has been tested on an {\bfseries{\href{https://www.hardkernel.com/shop/odroid-m1s-with-8gbyte-ram-io-header/}{\texttt{ ODROID M1S}}}} companion computer interfaced with a {\bfseries{\href{https://docs.px4.io/main/en/flight_controller/pixhawk6c.html}{\texttt{ Pixhawk 6c}}}}. The companion computer runs {\bfseries{Ubuntu 20.\+04 Linux}}.
